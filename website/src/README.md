# Website Source

This folder is the future component/application layer.

Current V1 pages are static HTML at the website root. Future implementation should migrate repeated layout, sections, forms, and content rendering into this source structure without changing the public routes.

## Structure

- `app/`: route and application shell planning.
- `components/`: reusable UI pieces.
- `data/`: canonical data used by pages and systems.
- `features/`: domain-specific feature modules.
- `lib/`: shared utilities and integrations.
- `styles/`: design tokens and future style modules.
- `types/`: shared TypeScript types.
